## ✨ 这是什么

一个给**体制内文秘 / 办公室 / 职能科室**用的公文写作助手。你不用再对着空白文档发愁“框架怎么立、亮点怎么写”"这个该写请示还是报告""行文方向对不对"——把要办的事、手头的素材、领导的意图丢给它，它按**党政机关公文处理规范**帮你把稿子立起来。

与"传统人工起草"和"普通 AI 写作"相比，它的定位是这样的：

| 维度 | 传统人工起草 | 普通 AI 写作 | 公文写作助手 |
|---|---|---|---|
| **文种 / 行文方向** | 靠经验，新手易把请示写成报告、对不相隶属机关误用请示 | 你说啥写啥，容易文种用错、方向混乱 | 内置「文种速判表」据意图定文种，**方向错先卡住**再动笔 |
| **一稿到成稿** | 找模板→套结构→逐段憋，反复返工 | 能快速出一版，但结构未必合规 | 大纲确认→逐段填，**一次到规范初稿** |
| **格式版式** | 红头/字号/署名日期手动排，易漏要素 | 排版全靠自己调 | 附「版式校验清单」逐项核对，**一键导出规范 Word** |
| **数据真实性** | 容易加错、手抖移个小数点 | 缺数据就编一个 | 未证实的数据一律留 `【待补：xx】` 占位，**绝不杜撰**文号/数字/人名 |
| **文风口径** | 因人而异，套话/口语化风险 | 像通用文章，公文风味不足 | 政府文风库 + 负面清单自查（政治性差错/套话/事实风险） |

## 🚀 使用方法

### 1️⃣ 30S安装

**① 一行命令（最快）**：

```bash
npx skills add https://github.com/onlyLT/wow-gongwen-writing --skill gongwen-writing
```

**② 让有 shell 权限的 AI Agent 帮你装（复制这段话直接发）**：

```text
帮我安装公文写作 skill。请把 https://github.com/onlyLT/wow-gongwen-writing 克隆到 ~/.claude/skills/ 目录下（克隆后该目录里应能看到 SKILL.md）
```

**③ 手动放进 skills 目录**：下载 / 克隆本仓库后，把整个目录放进你的 agent 的 skills 加载路径即可——如Claude Code 放进 `~/.claude/skills`目录下，重启后自然触发；其他 Agent Skills 兼容平台放进各自对应的路径。

> 核心写作能力**零依赖**，任何 Agent Skills 兼容 harness 都能跑（导出 Word 是可选能力，见下）。安装后技能触发名为 `gongwen-writing`。

### 2️⃣ 三种常见用法

```
① 直接说需求   →  "帮我写一份关于开展安全生产大检查的通知"
② 给一堆要点   →  "把这些要点写成给市政府的请示：……"
③ 拿稿子来改   →  "这份报告帮我按公文规范改一下 / 润色一下"
```

### 3️⃣ 核心工作流：先出大纲，再逐段填

> 默认不会一上来就冲一篇成稿给你，而是走一套**可控、可回头**的流程——每一步你都能叫停、纠偏。

```
①取料        问一轮（最多3问）：文种 / 发文机关·行文方向 / 事由诉求 / 关键数据
   ↓
②定文种      按「文种速判表」定文种 → 核对结构骨架与易错点（方向错先卡住）
   ↓
③出大纲      先给"标题 + 分层骨架" ⏸️ 停下等你确认
   ↓
④填成稿      按骨架展开，套用政府文风，对照负面清单自查
   ↓
⑤版式校验    核对 GB/T 9704 版式要素，随稿附「校验清单」
   ↓
⑥导出Word    (按需) 生成接近标准版式的 .docx
```

### 4️⃣ 一键导出规范版式 Word

需要 `.docx` 时，成稿会被组织成 JSON 规格，喂给内置导出器，产出 **GB/T 9704 近似版式**的公文——红头/报头、发文字号、仿宋三号正文、黑体标题、署名日期，一应俱全。

```bash
pip install -r scripts/requirements.txt      # 首次装依赖 python-docx
python scripts/export_docx.py spec.json       # 生成 Word
```

> 没装 Python / python-docx 也不影响——会**自动降级**为交付 markdown 成稿 + 版式说明，不阻断主体交付。

### 🎤 场景一 · 领导交流发言稿（事务性）★ 真实全文

> **输入**：一句话——"写一份在全省会议上的乡村振兴交流发言稿"。
> **输出**：一篇"三个坚持"结构、讲话口吻、数据处全部留占位的交流发言稿，直接可套本地数据。

<details>
<summary>📖 点击展开成稿片段</summary>

> **在全省【待补：会议名称】上的交流发言**
> ——【待补：县名】县农业农村局
>
> 　　尊敬的各位领导，同志们：
> 　　大家好！很荣幸代表**【待补：县名】**县作交流发言……2025年，全县农林牧渔业总产值达**【待补】**亿元、增长**【待补】**%，农村居民人均可支配收入达**【待补】**元……借此机会，我把我县的做法向各位领导和兄弟县市作个汇报，概括起来是"三个坚持"：
>
> 　　**一、坚持产业为基，做好"土特产"文章，让乡村振兴有奔头**
> 　　产业振兴是乡村振兴的重中之重……一是抓规模强基地。二是抓品牌提身价。三是抓融合延链条……把"土特产"三个字琢磨透，小产业也能做成大文章。
>
> 　　**二、坚持塑形铸魂，建设和美乡村，让农村环境有看头**
> 　　我们学习运用"千万工程"经验，一张蓝图绘到底……实现了从"一村美"向"村村美、片片美"的跃升……
>
> 　　**三、坚持富民导向，创新联农带农机制，让农民口袋有赚头**
> 　　检验乡村振兴成效，最终要看农民的钱袋子……
>
> 　　我的发言完毕，不当之处，请各位领导和同志们批评指正。谢谢大家！

</details>

👉 **看点**：讲话稿的"起—承—结"完整，三个分论点句式排比、朗朗上口；所有数据一律留 `【待补】` 待你填本地口径，涉及政治表述处还贴心加了「【建议核对本省惯用表述】」提醒。📎 [下载完整 docx](./examples/示例-乡村振兴交流发言稿.docx)

> **输入要点**：国庆中秋假期，部署安全生产工作。
> **skill 自动把关**：布置工作、事项分条、结尾"特此通知"。

<details>

> **××市人民政府办公室关于做好2026年国庆中秋假期安全生产工作的通知**
> 各县（市、区）人民政府，市政府各部门、直属机构：
>
> 　　国庆中秋假期临近，人流物流集中、生产经营活动频繁，安全生产风险叠加。为有效防范化解各类安全风险，现就做好假期安全生产工作通知如下。
>
> 　　一、**压紧压实责任**。各地各部门要严格落实"党政同责、一岗双责"……
> 　　二、**突出重点领域**。聚焦道路交通、消防、燃气、旅游、建筑施工等重点行业……
> 　　三、**强化应急值守**。严格执行领导带班和24小时值班制度……
>
> 　　特此通知。

</details>

👉 **看点**：下行文口吻、任务分条可落地、主送机关表述规范（"各县（市、区）人民政府"顿号连接）。

### 🤝 场景四 · 函（平行文，商洽不相隶属机关）

> **输入要点**：向同级不相隶属的交通部门，商请提供某项目用地规划意见。
> **skill 自动把关**：对不相隶属机关**用"函"而非"请示"**——这是最常见的行文硬伤之一。

<details>
<summary>📖 点击展开成稿片段</summary>

> **××市自然资源和规划局关于商请提供××项目用地控制性详细规划意见的函**
> ××市交通运输局：
>
> 　　我局正在办理××项目建设用地规划审查，该项目用地涉及贵局主管的××通道控制范围。为确保规划衔接、依法审批，现商请贵局就该项目用地的控制性详细规划及相关控制要求提供书面意见。
>
> 　　请于**【待补：时限】**前函复为盼。

</details>

👉 **看点**：平行文用商洽语气（"商请""为盼"），既不上行也不下行——把行文方向摆正，是公文规范的第一道关。

## 📊  规范把关


### 内建的规范把关（每次成稿自动过一遍）

- ✅ **行文方向**：上行 / 下行 / 平行文规则，请示"一文一事、单主送、需批复"，报告"不得夹带请示事项"
- ✅ **版式要素**：标题、主送、发文字号、成文日期、附件说明、抄送、印章位（依据 GB/T 9704-2012）
- ✅ **负面清单**：政治性差错、格式硬伤、行文规则违规、文风套话过度、事实/数据风险，逐条自查
- ✅ **风险提示**：涉密、重大决策、对外表述一律提示"须经本单位核稿人/保密审查终审"，不代替把关

## ⚠️ 使用须知

- 本 skill 为**政府口径**通用写作助手，涉密、重大决策、对外表述、党言党语专稿等，**须经本单位核稿人 / 保密审查终审**，工具不代替把关。

- 生成内容仅供起草参考，正式行文前请按本单位流程审核。
