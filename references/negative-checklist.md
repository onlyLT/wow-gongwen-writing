# 负面清单（成稿前逐条自查）

成稿后按五类逐条过一遍，把命中项列入随稿的「校验清单」。**政治性、格式硬伤、行文规则**为必检硬项。

## 一、政治性与表述风险（硬项）
- [ ] 引用党中央/上级重要论述、决策部署是否**准确完整**，无断章取义、无错字漏字？拿不准处已留 `【待核对原文】`？
- [ ] 重大方针、政治定性、对外口径是否**未擅自发挥**，已提示核稿人/宣传/保密把关？
- [ ] 涉密信息是否按密级处理，公开发布件是否已剔除不宜公开内容？
- [ ] 机构、职务、人名、地名、排序是否规范准确？

## 二、文种与行文规则（硬项）
- [ ] 文种是否选对（请示 vs 报告、通知 vs 意见、函 vs 请示）？
- [ ] **请示**：是否一文一事？是否单一主送、未多头？是否未夹带汇报？结尾有请求语？
- [ ] **报告**：是否未夹带请示/请求事项？
- [ ] **批复**：是否针对具体请示、态度明确？
- [ ] **函**：对象是否为不相隶属机关（否则应改文种）？
- [ ] 上行文是否标注了**签发人**？下行文是否未误标签发人？
- [ ] 是否越级行文（无特殊事由不越级）？抄送是否恰当（上行文不抄下级）？

## 三、格式与版式硬伤（硬项）
- [ ] 标题=发文机关+事由+文种，无多余标点，事由未与文种重复？
- [ ] 发文字号：`代字〔年份〕序号`，六角括号、年份全称、序号无虚位无"第"？
- [ ] 主送机关顶格、规范称谓？
- [ ] 成文日期：阿拉伯数字全称、月日无虚位、与署名一致？
- [ ] 附件说明写法规范（`附件：1.×××`，末尾无标点、无书名号）？
- [ ] 层次序数规范（`一、（一）1.（1）`），未越层？
- [ ] 印章/署名安排正确（纪要、部分函可不盖章）？

## 四、内容与逻辑
- [ ] 缘由—事项—结语三段是否齐全、逻辑连贯？
- [ ] 任务是否落到**牵头/责任单位 + 时限 + 标准**（方案、意见、通知类）？
- [ ] 数据是否有来源、可核、前后口径一致（单位、同比/完成率不矛盾）？
- [ ] 总结/汇报是否既讲成效也讲问题，打算与问题对应？
- [ ] 无 `【待补】` 遗漏未提示用户补齐？

## 五、文风与文字
- [ ] 是否避免空话套话堆砌（"高度重视/切实加强"须有实招支撑）？
- [ ] 无口语、网络用语、生造词进入正式公文？
- [ ] 无长句缠绕、"的的不休"、一段到底？
- [ ] 数字/计量书写规范（统计用阿拉伯数字，成词用汉字，全文统一）？
- [ ] 简称是否首次全称后统一使用？
- [ ] 结尾语与文种匹配？

---

**输出「校验清单」示例格式：**

```
【格式校验】
✔ 标题三要素齐全
✔ 发文字号规范（武政发〔2026〕5号）
⚠ 请示为上行文，需补标签发人 → 【待补：签发人】
【行文规则】✔ 一文一事、单主送
【风险提示】重大表述建议核稿人终审
【待补清单】①签发人 ②截至6月底具体投资数据
```
